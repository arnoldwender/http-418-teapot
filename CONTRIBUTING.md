# Contributing to HTTP 418 — I'm a Teapot

Welcome, fellow tea enthusiast! We're glad you want to help improve the world's most important HTTP status code experience. Pull up a chair, steep some leaves, and let's brew something great together.

## Getting Started

```bash
git clone https://github.com/arnoldwender/http-418-teapot.git
cd http-418-teapot
npm install
npm run dev
```

Open [http://localhost:5173](http://localhost:5173) and make sure you can see a teapot. If you see a coffee maker, something has gone terribly wrong.

## Submitting a Pull Request

1. Fork the repo and create your branch from `main`
2. Make your changes
3. Make sure the app still builds (`npm run build`)
4. Open a PR with a clear description of what you changed and why
5. Wait for review — we'll get to it between tea breaks

## Code Style

- **TypeScript** — no `any` unless you have a really good excuse (you probably don't)
- **React** functional components with hooks
- Keep the humor smart, not childish — think British dry wit, not slapstick
- If you add a new tea blend, it must have a corresponding HTTP status code. We don't make the rules. Actually we do, but still.

## Bug Reports

Open an issue with:
- What you expected to happen
- What actually happened
- Your browser and OS
- Whether you were drinking tea or coffee at the time (important diagnostic info)

## Feature Requests

We love new ideas! Open an issue and describe your feature. Bonus points if it's tea-related. Negative points if it involves coffee.

## The Golden Rule

This is a joke app about a joke RFC. Keep it fun, keep it clever, and never — under any circumstances — serve coffee.
